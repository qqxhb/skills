# Reply Template

Use this template for GitHub issue replies. Keep comments concise and evidence-backed.

## Question Reply

```markdown
Thanks for the question.

Based on the current repository behavior, ...

Evidence:
- ...

If you are using a different version or configuration, please share ...
```

## Needs-Info Reply

```markdown
Thanks for reporting this. I need a bit more information to verify it:

- ...
- ...

The smallest reproduction case would be especially helpful.
```

## Already-Fixed Reply

```markdown
Thanks for the report. This appears to be fixed on the current default branch.

Evidence:
- ...

If you can still reproduce it on version/commit ..., please share:
- ...
```

## Bug Fix PR Reply

```markdown
Thanks for the report. I opened a PR that addresses this:

- PR: ...
- Summary: ...
- Tests: ...

Remaining risk:
- ...
```

## Duplicate Reply

```markdown
Thanks for the report. This looks related to or covered by ...

The overlap is:
- ...

If your case differs, please add:
- ...
```

## Tone Rules

- Be direct and respectful.
- Do not blame the reporter.
- Do not over-promise timelines or release versions.
- Do not claim maintainer decisions unless confirmed.
- Include reproduction or verification details when discussing behavior.
